## 왜 이 커리큘럼인가?

- **모듈형 학습**: 기초 → 스프링 → 데이터/설계 → 운영/모니터링 → Q&A 흐름으로 계속 확장
- **실습 중심**: 매회 작은 산출물(코드/문서/테스트)을 남기도록 설계
- **지속 업데이트**: 회고와 추가 학습 주제를 계속 보강

## 커리큘럼 구조

1. [기초 다지기](/learning/career/backend-foundation-phase/) — 언어/알고리즘/CS 기본기
2. [스프링 핵심](/learning/career/backend-spring-core-phase/) — 스프링/테스트/JPA
3. [데이터 & 시스템 설계](/learning/career/backend-data-system-phase/) — DB 튜닝, Redis, 설계 문제
4. [운영 & 배포/모니터링](/learning/career/backend-ops-observability-phase/) — Docker, CI/CD, 모니터링, 클라우드
5. [핵심 개념 Q&A 모음](/learning/career/backend-qna-phase/) — 자주 묻는 질문과 답변 정리

## 학습 루틴 (순환형)

- **평일**: 2시간 (개념 1h + 문제풀이/실습 1h)
- **주말**: 4~6시간 (프로젝트 기능 추가, 회고/블로그 포스팅)
- **리뷰**: 2~4주마다 학습 로그 점검, 체크리스트 업데이트

## 산출물 체크리스트

- [ ] 꾸준히 **블로그 포스트 작성** (학습/트러블슈팅 정리)
- [ ] **레포 README**에 아키텍처 다이어그램, 결정 근거 기록
- [ ] **테스트 커버리지**: 핵심 서비스/레포지토리 단위 테스트 추가
- [ ] **성능/장애 기록**: 부하 테스트 지표, 장애 대응 시나리오 메모

## 추천 자료 (단권화)

- 언어/프레임워크: Effective Java, 김영한 스프링 로드맵, Java Concurrency in Practice
- 데이터/설계: Real MySQL 8.0, 데이터 중심 애플리케이션 설계, System Design Interview v1/v2
- 운영: 쿠버네티스 인 액션, 구글 SRE, Prometheus/Grafana 공식 문서

## 바로 시작하기

- 기초부터 순서대로 보려면 → [기초 다지기](/learning/career/backend-foundation-phase/)
- 스프링/데이터/운영 단계를 순서대로 보려면 → 위 1~4단계를 따라가세요
